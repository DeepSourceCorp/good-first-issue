# Welcome! 👋🏼

**Good First Issue** is an initiative to curate beginner-friendly issues from popular projects, helping developers make their first open-source contribution quickly and confidently.

Open-source maintainers are always looking to get more people involved, but new developers often find it challenging to get started. We believe that solving simple, beginner-friendly issues lowers the barrier to entry and builds confidence for future contributions.

---

## 🚀 Adding a New Project

You're welcome to add a new project to Good First Issue. We encourage contributions from all kinds of projects — old or new, big or small.

### ✅ Requirements

To maintain the quality of projects listed, please ensure your repository meets the following criteria:

| Requirement        | Description                                                                                                        |
| ------------------ | ------------------------------------------------------------------------------------------------------------------ |
| Good First Issues  | At least 3 open issues with beginner-friendly labels (`good first issue`, `beginner`, `easy`, `help wanted`, etc.) |
| Contributors       | At least 10 contributors                                                                                           |
| README.md          | Includes clear setup instructions                                                                                  |
| CONTRIBUTING.md    | Provides guidelines for new contributors                                                                           |
| Active Maintenance | Shows recent commits and activity                                                                                  |
| License            | Has a valid open-source license                                                                                    |

Once your submission is reviewed and approved, it will be added to https://goodfirstissue.dev/

## 🤝 Contributing

Want to contribute to this project? Follow these steps:

1. Fork the repository
2. Clone your fork
3. Create a new branch
4. Make your changes
5. Commit and push your changes
6. Open a Pull Request

For detailed instructions, refer to **CONTRIBUTING.md**

---

## 💡 Tips for First-Time Contributors

* Start with small changes (documentation, typos, formatting)
* Always read the contributing guidelines
* Keep your PRs simple and focused
* Write clear commit messages
